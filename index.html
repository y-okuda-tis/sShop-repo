// features

主な機能

ECサイトとして必要な機能を一通り実装。学習・参照・拡張のベースとして活用できます。

🔐

ログイン・アカウント管理

メール＋パスワード認証、ログイン失敗5回でアカウントを30分ロックするセキュリティ機能付き。

🛍️

買い物かご

商品の追加・個数変更・全削除をセッションで管理。表示時に最新在庫と突き合わせて自動調整。

📦

注文フロー

届け先入力 → 支払方法選択 → 確認 → 完了の4ステップ。確定時に二重の在庫チェックを実施。

🗂️

商品・カテゴリ管理

管理者が商品の登録・更新・削除、画像アップロードをGUI操作で実施。ソフトデリートで履歴を保持。

👥

会員管理

一般会員は自身の情報を管理。管理者は権限に応じて他会員を閲覧・編集・削除できます。

📋

注文履歴・管理

一般会員は自分の注文一覧を確認。管理者は全注文を日付降順で閲覧・詳細確認できます。

🔍

商品一覧・検索

カテゴリフィルタと「新着順 / 売れ筋順」の2ソートに対応。注文数を集計したJPQLクエリを実装。

✅

カスタムバリデーション

メールアドレス重複・カテゴリ名重複・商品名重複を独自アノテーション＋バリデータで実装。

🛡️

4段階アクセス制御

Filterチェーンで未ログイン・一般会員・運用管理者・システム管理者のアクセスを制御。

// architecture

設計のポイント

標準的なMVC構造に加え、独自の工夫が随所に施されています。

PRG Pattern

二重送信対策

POSTの完了後は必ずリダイレクト（GET）で完了画面を表示し、ブラウザバックによる再送信を防止。

Session管理

入力フォームの保持

入力 → 確認 → 完了の多段フローで、フォーム情報とバリデーションエラーをセッションで引き渡す。

在庫管理

2段階在庫チェック

確認画面表示時と注文確定時の2回チェック。タイムラグによる在庫超過注文を防ぐ設計。

注文履歴

注文時単価の保存

OrderItemに購入時の単価を保存。後から価格が変わっても、注文時の正確な金額を参照可能。

Soft Delete

論理削除

会員・商品・カテゴリはDeleteFlagで論理削除。データは残るため注文履歴との整合性を保てる。

Bean Conversion

BeanToolsサービス

Entity ↔ Form ↔ Bean 間のコピー処理を1クラスに集約。コントローラのコードを簡潔に保つ。
